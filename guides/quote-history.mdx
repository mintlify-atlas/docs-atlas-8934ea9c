The quote history interface provides centralized management of all saved quotes. Access your complete quote archive, reopen quotes for editing, create duplicates, or delete old records.

## Accessing Quote History

The quote history is available at `quotes.html` and requires Firebase authentication.

### From the Main App

1. Click the **hamburger menu** (☰) in the top-right corner of `index.html`
2. Click **Ver historial** (View history)
3. Sign in if prompted

Alternatively, navigate directly to `quotes.html`.

### From the Client Directory

Click the **Historial** button in the top navigation bar of `clients.html`.

<Note>
  Quote history requires Firebase Authentication. Quotes are stored per user, so you'll only see your own quotes when logged in.
</Note>

## Authentication

### Login Flow

If not authenticated, the auth card is displayed:

```javascript
// Login handler (quotes.js:33-51)
async function doLogin(evt) {
  evt.preventDefault();
  const email = $('loginEmail').value.trim();
  const password = $('loginPassword').value;
  
  if (!email || !password) {
    showStatus('Completa correo y contraseña.', true);
    return;
  }
  
  try {
    await window.FirebaseAuthService.signIn(email, password);
    showStatus('Sesión iniciada.', false);
    await bootstrap(); // Load quotes
  } catch (e) {
    const msg = window.FirebaseAuthService.mapAuthError(e.message);
    showStatus(msg, true);
  }
}
```

**Authentication Service** is provided by `firebase-auth.js`, which handles:
- Sign in with email/password
- Token refresh
- Session persistence
- Error mapping to user-friendly Spanish messages

### Session Display

Once authenticated, the session info shows the logged-in email:

```javascript
// Update UI based on auth state (quotes.js:27-31)
function setAuthUi(logged, email) {
  $('authCard').classList.toggle('hidden', logged);
  $('quotesListCard').classList.toggle('hidden', !logged);
  $('sessionInfo').textContent = logged ? `Sesión: ${email || ''}` : 'Sin sesión';
}
```

<Tip>
  Firebase Authentication tokens are automatically refreshed by `firebase-auth.js`. Sessions persist across page reloads using browser storage.
</Tip>

## Quote List Display

### Loading Quotes

Quotes are fetched from Firebase on page load:

```javascript
// Load quotes (quotes.js:140-151)
async function loadQuotes() {
  const { user, idToken } = await requireAuthContext();
  const list = await window.QuotesRepo.listQuotes(user.uid, idToken);
  const container = $('quoteList');
  
  if (!list.length) {
    container.innerHTML = '<div class="quote-item">No hay cotizaciones guardadas.</div>';
    return;
  }
  
  container.innerHTML = list.map(quoteItemTemplate).join('');
}
```

Quotes are returned sorted by `updatedAt` timestamp (most recent first) from `QuotesRepo.listQuotes()`.

Each quote is rendered as a card with key information:

    ```javascript
    // Duplicate quote (quotes.js:160-186)
    async function duplicateQuote(id) {
      const { user, idToken } = await requireAuthContext();
      const quote = await window.QuotesRepo.getQuote(user.uid, idToken, id);
      if (!quote) throw new Error('No se encontró para duplicar.');
      
      // Extract year from original quote number
      const targetYear = getQuoteYearFromNumber(quote.quoteNumber);
      
      // Find highest sequence number for that year
      const allQuotes = await window.QuotesRepo.listQuotes(user.uid, idToken);
      const maxSeq = (allQuotes || []).reduce((max, item) => {
        const seq = parseQuoteSequenceByYear(item.quoteNumber, targetYear);
        return seq > max ? seq : max;
      }, 0);
      
      // Generate new quote number
      const nextQuoteNumber = buildQuoteNumber(targetYear, maxSeq + 1);
      
      // Create clone
      const clone = {
        ...quote,
        id: undefined,              // New ID will be generated by Firebase
        activeQuoteId: null,
        status: 'draft',
        quoteNumber: nextQuoteNumber,
        createdAt: Date.now(),
        updatedAt: Date.now()
      };
      
      await window.QuotesRepo.saveQuote(user.uid, idToken, clone, null);
      showStatus('Cotización duplicada.', false);
      await loadQuotes(); // Refresh list
    }
    ```
    
    **Use Cases:**
    - Create similar quotes for different clients
    - Reuse equipment/service combinations
    - Generate multiple options with slight variations
    
    <Tip>
      Duplicate quotes use the same year as the original but get the next available sequence number. This keeps related quotes together chronologically.
    </Tip>

## Quote Status Tracking

Quotes can have two status values:

### Status Values

<Tabs>
  <Tab title="draft">
    **Spanish label**: "Borrador"
    
    **Meaning**: Quote is saved but not yet finalized or issued to client
    
    **Set when:**
    - Clicking "Guardar borrador" (Save draft) in the editor
    - Duplicating a quote (duplicates are always drafts)
    - Auto-saving work in progress
    
    **Use cases:**
    - Quotes you're still working on
    - Quotes awaiting client information
    - Alternative options not yet sent to client
  </Tab>
  
  <Tab title="issued">
    **Spanish label**: "Emitida"
    
    **Meaning**: Quote has been finalized and issued to the client
    
    **Set when:**
    - Manually changed in code (no UI button for this currently)
    - After generating and sending PDF to client
    
    **Use cases:**
    - Quotes that have been delivered to clients
    - Historical record of sent quotes
    - Tracking which quotes are outstanding
    
    <Note>
      Currently, there's no built-in UI to change status from draft to issued. You can add this functionality by extending the quote editor or history interface.
    </Note>
  </Tab>
</Tabs>

### Status Display

```javascript
// Status label formatting (quotes.js)
function statusLabel(status) {
  if (status === 'issued') return 'Emitida';
  if (status === 'draft') return 'Borrador';
  return status || 'Borrador';
}
```

Status is displayed in the quote metadata line:

```

```

## Quote Number Parsing

The history interface includes utilities for parsing quote numbers:

### Year Extraction

```javascript
// Extract year from quote number (quotes.js:105-115)
function getQuoteYearFromNumber(quoteNumber) {
  const normalized = String(quoteNumber || '').trim().toUpperCase();
  
  // Current format: CL-2026-0001
  const currentFmt = normalized.match(/^CL-(\d{4})-(\d{4,})$/);
  if (currentFmt) return currentFmt[1];
  
  // Legacy format: COT-20260311-0001
  const legacyFmt = normalized.match(/^COT-(\d{8})-(\d{4,})$/);
  if (legacyFmt) return legacyFmt[1].slice(0, 4);
  
  return String(new Date().getFullYear());
}
```

### Sequence Parsing

```javascript
// Parse sequence number for specific year (quotes.js:117-134)
function parseQuoteSequenceByYear(quoteNumber, year) {
  const normalized = String(quoteNumber || '').trim().toUpperCase();
  
  // Current format: CL-2026-0001
  const currentFmt = normalized.match(/^CL-(\d{4})-(\d{4,})$/);
  if (currentFmt) {
    if (String(year) !== currentFmt[1]) return 0;
    return Number(currentFmt[2] || 0) || 0;
  }
  
  // Legacy format: COT-20260311-0001
  const legacyFmt = normalized.match(/^COT-(\d{8})-(\d{4,})$/);
  if (legacyFmt) {
    const legacyYear = legacyFmt[1].slice(0, 4);
    if (String(year) !== legacyYear) return 0;
    return Number(legacyFmt[2] || 0) || 0;
  }
  
  return 0;
}
```

### Quote Number Generation

```javascript
// Build new quote number (quotes.js:136-138)
function buildQuoteNumber(year, sequence) {
  return `CL-${year}-${String(Math.max(1, Number(sequence) || 1)).padStart(4, '0')}`;
}
```

**Examples:**
- Year: 2026, Sequence: 1 → `CL-2026-0001`
- Year: 2026, Sequence: 42 → `CL-2026-0042`
- Year: 2026, Sequence: 1234 → `CL-2026-1234`

<Accordion title="Why parse quote numbers?">
  Quote number parsing is essential for:
  
  1. **Duplicate detection**: When duplicating, find the highest sequence for the year
  2. **Year isolation**: Keep sequences separate by year (resets each January 1)
  3. **Legacy support**: Handle old quote format (COT-YYYYMMDD-####) transparently
  4. **Sorting**: Can sort quotes by parsed year and sequence
  
  This ensures duplicated quotes get proper sequential numbers and old quotes remain compatible.
</Accordion>

## Quote Repository API

Quote history relies on `QuotesRepo` from `quotes-repo.js`:

### Core Methods

<Tabs>
  <Tab title="listQuotes">
    ```javascript
    async function listQuotes(userId, idToken) {
      const endpoint = `${FIREBASE_REST_API_BASE_URL}/users/${userId}/quotes.json?auth=${idToken}`;
      const response = await fetch(endpoint);
      const data = await response.json();
      
      if (!data) return [];
      
      // Convert object to array
      const quotes = Object.entries(data).map(([id, quote]) => ({
        ...quote,
        id: id
      }));
      
      // Sort by updatedAt descending (newest first)
      return quotes.sort((a, b) => 
        Number(b.updatedAt || 0) - Number(a.updatedAt || 0)
      );
    }
    ```
    
    Returns array of all quotes for the user, sorted by most recent.
  </Tab>
  
  <Tab title="getQuote">
    ```javascript
    async function getQuote(userId, idToken, quoteId) {
      const endpoint = `${FIREBASE_REST_API_BASE_URL}/users/${userId}/quotes/${quoteId}.json?auth=${idToken}`;
      const response = await fetch(endpoint);
      return await response.json();
    }
    ```
    
    Returns single quote object by ID, or `null` if not found.
  </Tab>
  
  <Tab title="saveQuote">
    ```javascript
    async function saveQuote(userId, idToken, quoteSnapshot, quoteId) {
      const id = quoteId || quoteSnapshot.id || generatePushId();
      const quote = {
        ...quoteSnapshot,
        id: id,
        updatedAt: Date.now()
      };
      
      if (!quote.createdAt) {
        quote.createdAt = Date.now();
      }
      
      const endpoint = `${FIREBASE_REST_API_BASE_URL}/users/${userId}/quotes/${id}.json?auth=${idToken}`;
      await fetch(endpoint, {
        method: 'PUT',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(quote)
      });
      
      return { ...quote };
    }
    ```
    
    Creates new quote or updates existing one. Returns saved quote object.
  </Tab>
  
  <Tab title="deleteQuote">
    ```javascript
    async function deleteQuote(userId, idToken, quoteId) {
      const endpoint = `${FIREBASE_REST_API_BASE_URL}/users/${userId}/quotes/${quoteId}.json?auth=${idToken}`;
      await fetch(endpoint, { method: 'DELETE' });
    }
    ```
    
    Permanently deletes quote from Firebase. No return value.
  </Tab>
</Tabs>

## Data Flow

### Complete Quote Lifecycle

1. **Creation**: User creates quote in main editor
2. **Save**: Click "Guardar borrador" → Calls `quote-persistence.js`
3. **Storage**: `QuotesRepo.saveQuote()` writes to Firebase
4. **Client linking**: If RUT provided, `ClientRepo.saveOrUpdateClient()` links quote to client
5. **History view**: User opens `quotes.html`
6. **Load**: `QuotesRepo.listQuotes()` fetches all quotes
7. **Display**: Quotes rendered with action buttons
8. **Edit**: User clicks "Abrir y editar" → Redirects with quote data in sessionStorage
9. **Update**: Editor loads quote, user modifies, clicks save → Updates existing quote in Firebase
