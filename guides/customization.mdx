CotizaClima is designed to be customizable for different HVAC businesses. This guide covers how to modify services, accessories, branding, and PDF content to match your business needs.

## Customizing Services

Services are defined in the `SERVICES` array in `app.js` (lines 5-12).

### Service Structure

```javascript
const SERVICES = [
  { 
    id: 'instalacion',           // Unique identifier
    name: 'Instalación nueva',   // Display name
    desc: 'Primera instalación del equipo',  // Description
    price: 100000,                // Base price in CLP
    icon: 'fa-hammer'             // Font Awesome icon class
  },
  // ... more services
];
```

### Modifying Existing Services

To change a service, edit its entry in the array:

<Tabs>
  <Tab title="Change Price">
    Update the `price` field:
    
    ```javascript
    { 
      id: 'instalacion', 
      name: 'Instalación nueva', 
      desc: 'Primera instalación del equipo', 
      price: 150000,  // Changed from 100000
      icon: 'fa-hammer' 
    }
    ```
    
    This sets the default/base price. Users can still adjust it per-quote in the service selection interface.
  </Tab>
  
  <Tab title="Change Name/Description">
    Update the `name` and `desc` fields:
    
    ```javascript
    { 
      id: 'instalacion', 
      name: 'Instalación profesional',  // New name
      desc: 'Instalación completa con garantía extendida',  // New description
      price: 100000, 
      icon: 'fa-hammer' 
    }
    ```
    
    These changes appear immediately in the service selection grid.
  </Tab>
  
  <Tab title="Change Icon">
    Update the `icon` field with any Font Awesome 6 class:
    
    ```javascript
    { 
      id: 'instalacion', 
      name: 'Instalación nueva', 
      desc: 'Primera instalación del equipo', 
      price: 100000, 
      icon: 'fa-screwdriver-wrench'  // Changed icon
    }
    ```
    
    Browse available icons at [fontawesome.com/icons](https://fontawesome.com/icons). Use the free solid icons (prefix `fa-`).
  </Tab>
</Tabs>

### Adding New Services

Add a new object to the `SERVICES` array:

```javascript
const SERVICES = [
  { id:'instalacion', name:'Instalación nueva', desc:'Primera instalación del equipo', price:100000, icon:'fa-hammer' },
  { id:'mantencion', name:'Mantención', desc:'Limpieza y revisión general', price:30000, icon:'fa-screwdriver-wrench' },
  { id:'cambio', name:'Cambio + instalación', desc:'Retiro equipo antiguo + nuevo', price:120000, icon:'fa-arrow-right-arrow-left' },
  { id:'reparacion', name:'Reparación', desc:'Diagnóstico y reparación de fallas', price:10000, icon:'fa-gear' },
  { id:'garantia', name:'Visita garantía', desc:'Revisión por garantía vigente', price:0, icon:'fa-shield' },
  { id:'presupuesto', name:'Solo presupuesto', desc:'Cotización sin instalación', price:0, icon:'fa-clipboard-list' },
  
  // NEW SERVICE
  { 
    id: 'recarga_gas', 
    name: 'Recarga de gas', 
    desc: 'Recarga de refrigerante R410 o R32', 
    price: 45000, 
    icon: 'fa-wind' 
  },
];
```

<Warning>
  **Important**: The `id` must be unique and use only lowercase letters, numbers, and underscores. Avoid spaces and special characters.
</Warning>

### Multi-Service vs Single-Service

By default, only `instalacion` and `mantencion` support multiple quantities. To enable this for other services, modify the `isMultiService()` function:

```javascript
// Current implementation (app.js:771-773)
function isMultiService(id) {
  return id === 'instalacion' || id === 'mantencion';
}

// Extended to include new service
function isMultiService(id) {
  return id === 'instalacion' || id === 'mantencion' || id === 'recarga_gas';
}
```

Multi-services display quantity controls (+/-) and allow adjusting the unit price inline.

### Removing Services

Simply delete the service object from the array:

```javascript
const SERVICES = [
  { id:'instalacion', name:'Instalación nueva', desc:'Primera instalación del equipo', price:100000, icon:'fa-hammer' },
  { id:'mantencion', name:'Mantención', desc:'Limpieza y revisión general', price:30000, icon:'fa-screwdriver-wrench' },
  // Removed: cambio, reparacion, garantia
  { id:'presupuesto', name:'Solo presupuesto', desc:'Cotización sin instalación', price:0, icon:'fa-clipboard-list' },
];
```

### Adding Content for New Services

When you add a new service, add its PDF content:

```javascript
const SERVICE_PDF_CONTENT = {
  instalacion: { /* ... */ },
  mantencion: { /* ... */ },
  reparacion: { /* ... */ },
  
  // Content for new service
  recarga_gas: {
    title: 'Recarga de Gas Refrigerante',
    relevant: [
      'Servicio de recarga de gas refrigerante para equipos con bajo rendimiento o pérdida de presión.',
      'Se utiliza refrigerante certificado R410A o R32 según especificaciones del fabricante.',
      'El servicio incluye:',
      '- Detección de fugas con equipo especializado',
      '- Evacuación del sistema si es necesario',
      '- Recarga precisa según capacidad del equipo',
      '- Verificación de presiones de trabajo',
      '- Prueba de funcionamiento post-recarga'
    ],
    extra: [
      'La recarga incluye hasta 1 kg de gas refrigerante.',
      'Cantidades adicionales se cobran por separado según tarifa vigente.',
      'Si se detectan fugas importantes, se cotizará la reparación antes de proceder.',
      'Garantía de 30 días en el servicio de recarga realizado.'
    ]
  }
};
```

<Note>
  PDF content is only included for services that have an entry in `SERVICE_PDF_CONTENT`. Services without content (like `presupuesto` or `garantia`) won't show additional sections in the PDF.
</Note>

### Content Rendering in PDF

The `relevant` and `extra` sections are rendered differently:

- **Relevant**: Each string becomes a paragraph with normal text
- **Extra**: Each string becomes a bullet point (prefixed with "- ")

The rendering is handled by `appendServiceInfoToPdf()` (app.js:1451-1504).

**Adding icon for new accessory:**

```javascript
const ACC_ICON_MAP = {
  // Existing mappings...
  
  // New accessory
  soporte_pared: 'fa-grip-vertical',  // Wall mount bracket
  valvula_servicio: 'fa-gauge',       // Service valve
  filtro_aire: 'fa-filter',           // Air filter

```

**Icon resolution order:**
1. Specific ID match (e.g., `kit_can_9k`)
2. Category fallback (e.g., `_cat_cañerias` for accessories with `categoria: "cañerias"`)
3. Default icon (`fa-box`) if no match

Implementation in `getAccIcon()` function (app.js:939-943):

```javascript
function getAccIcon(acc) {
  return ACC_ICON_MAP[acc.id]
    || ACC_ICON_MAP['_cat_' + (acc.categoria || '')]
    || 'fa-box';
}
```

### PDF Logo

The logo is defined as a base64-encoded PNG in `PDF_LOGO_BASE64` constant (app.js:1392):

```javascript
const PDF_LOGO_BASE64 = 'iVBORw0KGgoAAAANSUhEUgAAB9AAAAZuCAYAAAAcj73IAAEAAElEQVR42uz9e6xtSX...';
```

**To change the logo:**

1. **Prepare your logo**:
   - Format: PNG (transparent background recommended)
   - Size: 200x200 pixels or similar square ratio
   - Keep file size reasonable (under 100KB)

2. **Convert to base64**:
   - Online tool: [base64-image.de](https://www.base64-image.de/)
   - Command line: `base64 -i logo.png`
   - Node.js: `fs.readFileSync('logo.png', 'base64')`

3. **Replace the constant**:
   ```javascript
   // Option 1: Base64 string only
   const PDF_LOGO_BASE64 = 'iVBORw0KGgoAAAANSUhEUg...';
   
   // Option 2: Full data URL
   const PDF_LOGO_BASE64 = 'data:image/png;base64,iVBORw0KGgoAAAANSUhEUg...';
   ```

4. The `normalizePngDataUrl()` function (app.js:1394-1408) handles both formats automatically.

<Tip>
  Use an optimized PNG with transparency. Tools like [TinyPNG](https://tinypng.com/) can reduce file size while maintaining quality.
</Tip>

## Firebase Configuration

Firebase connection is configured in `FIREBASE_CONNECTION` object (app.js:64-68):

```javascript
const FIREBASE_CONNECTION = {
  databaseURL: 'https://cotizaciones-app-ece65-default-rtdb.firebaseio.com/',
  node: 'air_conditioners',  // Equipment data node
  authToken: ''               // Optional auth token for read access
};
```

### Changing Firebase Project

1. **Create new Firebase project** at [console.firebase.google.com](https://console.firebase.google.com)
2. **Enable Realtime Database**
3. **Configure database rules**:
   ```json
   {
     "rules": {
       "air_conditioners": {
         ".read": true,
         ".write": false
       },
       "accessories": {
         ".read": true,
         ".write": false
       },
       "users": {
         "$uid": {
           ".read": "auth.uid === $uid",
           ".write": "auth.uid === $uid"
         }
       }
     }
   }
   ```
4. **Update configuration**:
   ```javascript
   const FIREBASE_CONNECTION = {
     databaseURL: 'https://your-project-default-rtdb.firebaseio.com/',
     node: 'air_conditioners',
     authToken: ''
   };
   ```

5. **Configure Firebase Authentication** in `firebase-auth.js`:
   ```javascript
   const FIREBASE_API_KEY = 'your-api-key';
   const FIREBASE_AUTH_DOMAIN = 'your-project.firebaseapp.com';
   ```

### Firebase REST API Base URL

If you need to change the REST API endpoint, update constants in:

- `quotes-repo.js`: `FIREBASE_REST_API_BASE_URL`
- `client-repo.js`: `FIREBASE_REST_API_BASE_URL`
- `ranking-repo.js`: `FIREBASE_REST_API_BASE_URL`

```javascript

const BRAND_ICON_DIR = 'assets/img';
```

   ```javascript
   const BRAND_ICON_OVERRIDES = {
   ```

**Logo search order:**
1. Check `{brand}.svg` in assets/img/
2. Check `{brand}.png`, `{brand}.webp`, etc.
3. Use overrides if specified
4. If all fail, logo is hidden (no broken image)

## Quote Number Format

Customize the quote number format by modifying these functions:

### Format Pattern

Current format: `CL-YYYY-####`

To change to a different pattern:

```javascript
// Current implementation (app.js:378-380)
function buildQuoteNumber(datePart, sequence) {
  return `CL-${datePart}-${String(Math.max(1, Number(sequence) || 1)).padStart(4, '0')}`;
}

// Custom format examples:

// Format: ABC-001234 (no year)
function buildQuoteNumber(datePart, sequence) {
  return `ABC-${String(Math.max(1, Number(sequence) || 1)).padStart(6, '0')}`;
}

// Format: 2026/0001 (slash separator)
function buildQuoteNumber(datePart, sequence) {
  return `${datePart}/${String(Math.max(1, Number(sequence) || 1)).padStart(4, '0')}`;
}

// Format: Q-20260001 (year + sequence combined)
function buildQuoteNumber(datePart, sequence) {
  return `Q-${datePart}${String(Math.max(1, Number(sequence) || 1)).padStart(4, '0')}`;
}
```

<Warning>
  If you change the quote number format, you must also update the parsing functions:
  - `parseQuoteSequence()` in app.js
  - `getQuoteYearFromNumber()` in quotes.js
  - `parseQuoteSequenceByYear()` in quotes.js
  
  Otherwise, quote duplication and history features won't work correctly.
</Warning>

### Year Part

Currently uses 4-digit year. To use year-month:

```javascript
// Current implementation (app.js:374-376)
function getQuoteDatePart(date = new Date()) {
  return String(date.getFullYear());
}

// Custom: Use YYYYMM format
function getQuoteDatePart(date = new Date()) {
  const year = date.getFullYear();
  const month = String(date.getMonth() + 1).padStart(2, '0');
  return `${year}${month}`;
}
// Results in: CL-202603-0001 for March 2026
```

The app uses three font families:

```css
/* In HTML head */
<link href="https://fonts.googleapis.com/css2?family=Cormorant+Garamond:wght@400;600;700&family=Lato:wght@300;400;700&family=DM+Mono:wght@400;500&display=swap" rel="stylesheet">
```
