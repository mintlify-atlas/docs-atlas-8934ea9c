This guide walks you through the complete process of creating a quote in CotizaClima, from selecting equipment to generating the final PDF.

## Quote Creation Workflow

CotizaClima uses a 5-step wizard to guide you through creating professional HVAC quotes:

<Steps>
  <Step title="Select Equipment">
    Choose air conditioning units by brand, model, and BTU capacity
  </Step>
  
  <Step title="Choose Service">
    Select installation, maintenance, repair, or other services
  </Step>
  
  <Step title="Add Accessories">
    Add optional accessories like piping kits, electrical components, or drainage kits
  </Step>
  
  <Step title="Enter Client Information">
    Fill in customer details including name, address, and contact information
  </Step>
  
  <Step title="Review and Generate PDF">
    Review pricing, apply discounts, and download the professional PDF quote
  </Step>
</Steps>

## Step 1: Selecting Equipment

### Search and Filter

The equipment selection screen provides multiple ways to find the right unit:

```javascript
// Equipment filtering is handled by the filterAC() function in app.js
function filterAC() {
  const search = document.getElementById('searchInput').value.toLowerCase();
  const brand = document.getElementById('filterBrand').value;
  const btu = parseInt(document.getElementById('filterBTU').value) || 0;
  const ref = document.getElementById('filterRef').value;
  
  filteredAC = allAC.filter(ac => {
    // Filter by search term (brand or model)
    if (search && !brandName.includes(search) && !modelName.includes(search)) return false;
    // Filter by brand
    if (brand && (ac.marca || '') !== brand) return false;
    // Filter by refrigerant type
    if (ref && ac.refrigerant !== ref) return false;
    // Filter by BTU capacity
    if (btu && !ac.capacities.find(c => c.btu_capacity === btu)) return false;
    return true;
  });
}
```

**Available Filters:**
- **Search bar**: Type brand name or model number
- **Brand dropdown**: Filter by manufacturer (Daitsu, IKA, Kendal, LG, Midea, TCL, Clark)
- **BTU capacity**: Filter by cooling capacity (9K, 12K, 14K, 18K, 22K, 24K, 36K)
- **Refrigerant type**: Filter by R32 or R410 refrigerant
- **Popular sorting**: Sort by most/least quoted equipment

### Selecting BTU Capacity and Quantity

Each equipment card displays available BTU capacities as buttons:

1. **Click a BTU button** to add one unit of that capacity
2. The card displays the current quantity (e.g., "12K x2")
3. Use **+** and **-** buttons to adjust quantity
4. The card shows the subtotal price including tax

<Note>
  Equipment prices are displayed with IVA (VAT) included. The `price_with_tax` field from the equipment data is used for all calculations.
</Note>

### Multi-Service Support

**Instalación** and **Mantención** support multiple quantities:

1. Click the service card to add one unit
2. Use the **+** and **-** buttons to adjust quantity
3. Modify the unit price directly in the inline input field
4. Installation quantity is automatically capped at the number of equipment units selected

```javascript
// Installation is preloaded based on equipment count (app.js:141-155)
function preloadInstallationsFromEquipment() {
  const equipCount = getEquipCount();
  if (equipCount === 0) {
    serviceQtyMap.instalacion = 0;
  } else if ((serviceQtyMap.instalacion || 0) === 0) {
    // Default: 1 installation per equipment
    serviceQtyMap.instalacion = equipCount;
  } else {
    // Cap installations at equipment count
    serviceQtyMap.instalacion = Math.min(serviceQtyMap.instalacion, equipCount);
  }
}
```

### Repair Service Customization

The **Reparación** service allows custom quantity and unit price:

1. Select the service card
2. Inline inputs appear for **Cantidad** (quantity) and **Valor unitario** (unit price)
3. The total updates automatically as you type
4. Default values: 1 unit at $10,000

### Service Notes

Add detailed observations in the **Observaciones del servicio** textarea:

- Describe special conditions
- Specify piping lengths beyond standard 3 meters
- Note access restrictions or challenges
- Add any custom terms

These notes appear in the PDF quote below the pricing table.

## Step 4: Entering Client Information

### Required Fields

Only two fields are mandatory:

- **Nombre completo** (Full name)
- **Dirección de instalación** (Installation address)

### Optional Fields

- **RUT**: Chilean tax ID (only displayed in PDF if provided)
- **Teléfono** (Phone)
- **Correo electrónico** (Email)
- **Ciudad** (City)
- **Región** (Region): Dropdown with all Chilean regions
- **Tipo de inmueble** (Property type): Casa, Departamento, Oficina, Local comercial, Bodega/Industrial
- **Notas adicionales**: Special instructions, floor number, access details

### GPS Address Autofill

The **Usar GPS** button automatically fills address and city using your current location:

```javascript
// GPS autofill implementation (app.js:1838-1908)
async function fillAddressFromGPS() {
  // 1. Request geolocation permission
  const position = await navigator.geolocation.getCurrentPosition();
  
  // 2. Reverse geocode using OpenStreetMap Nominatim API
  const { latitude, longitude } = position.coords;
  const reverseUrl = `https://nominatim.openstreetmap.org/reverse?lat=${latitude}&lon=${longitude}&format=jsonv2`;
  const response = await fetch(reverseUrl);
  const data = await response.json();
  
  // 3. Parse address components
  const address = data.address;
  const streetAddress = buildStreetAddress(address); // Road + number + suburb
  const city = pickCityFromAddress(address); // City/town/village
  const region = mapRegionFromState(address.state); // Map to Chilean regions
  
  // 4. Fill form fields
  if (streetAddress) addressInput.value = streetAddress;
  if (city) cityInput.value = city;
  if (region) regionSelect.value = region;
}
```

<Note>
  GPS autofill requires browser location permission and an internet connection. It uses OpenStreetMap's Nominatim API for reverse geocoding.
</Note>

## Step 5: Reviewing and Generating PDF

### Summary Review

The final step displays a comprehensive summary:

- **Quote information**: Quote number, date, validity (15 days)
- **Client details**: All entered information
- **Equipment**: Model, capacity, refrigerant, price
- **Services**: Type, quantity, unit price, total
- **Accessories**: Name, quantity, subtotal for each
- **Pricing**: Subtotal, discount, grand total

### Adjusting Prices

You can edit prices before generating the PDF:

1. **Equipment price**: Editable input field shows total equipment price
2. **Service price**: Editable input field shows total service price
3. **Discount**: Enter a percentage (0-100%)
   - Displays discount amount as "−$XX.XXX"
   - Automatically recalculates total

```javascript
// Price recalculation (app.js:1118-1131)
function recalcTotal() {
  const equipPrice = parseFloat(document.getElementById('sumEquipPrice').value) || 0;
  const servicePrice = parseFloat(document.getElementById('sumServicePrice').value) || 0;
  let extrasTotal = 0;
  ACCESSORIES.forEach(acc => {
    extrasTotal += (accQty[acc.id]||0) * acc.price;
  });
  const subtotal = equipPrice + servicePrice + extrasTotal;
  const discPct = parseFloat(document.getElementById('discountPct').value) || 0;
  const discAmt = subtotal * discPct / 100;
  const total = subtotal - discAmt;
  // Update display
}
```

## Quote Number Format

Quote numbers follow the format: `CL-YYYY-####`

Example: `CL-2026-0001`

- `CL`: Company prefix
- `YYYY`: Current year
- `####`: Sequential number (resets each year)

The sequence counter is stored in:
1. Firebase (if authenticated): Synchronized across devices
2. LocalStorage (fallback): `cotiz_quote_counter_YYYY`

```javascript
// Quote number generation (app.js:374-461)
async function generateQuoteNumber() {
  const d = new Date();
  const datePart = getQuoteDatePart(d); // Returns year as string
  
  // Local fallback
  const localNext = readLocalQuoteCounter(datePart) + 1;
  writeLocalQuoteCounter(datePart, localNext);
  setQuoteNumberValue(buildQuoteNumber(datePart, localNext));
  
  // Firebase sync (if available)
  try {
    const firebaseNext = await getNextQuoteSequenceFromFirebase(datePart);
    if (firebaseNext) {
      writeLocalQuoteCounter(datePart, firebaseNext);
      setQuoteNumberValue(buildQuoteNumber(datePart, firebaseNext));
    }
  } catch (e) {
    // Keep local counter
  }
}
```

<Note>
  Legacy format `COT-YYYYMMDD-####` is still supported for backward compatibility but new quotes use the simplified `CL-YYYY-####` format.
</Note>
