CotizaClima is a static web application that runs entirely in the browser. No backend server required. Deploy to any web server, CDN, or local environment.

## Quick Install

<Steps>
  <Step title="Download the Files">
    Clone or download the CotizaClima repository:

    ```bash
    git clone https://github.com/your-org/cotizaclima.git
    cd cotizaclima
    ```

    The application consists of:

    ```bash File Structure
    cotizaclima/
    ├── index.html              # Main application
    ├── quotes.html             # Quote history page
    ├── clients.html            # Client management page
    ├── app.js                  # Core application logic (652KB)
    ├── firebase-auth.js        # Authentication service
    ├── quotes-repo.js          # Quote persistence
    ├── client-repo.js          # Client database
    ├── ranking-repo.js         # Equipment popularity tracking
    ├── quote-persistence.js    # Local storage utilities
    ├── quotes.js               # History page logic
    ```

  <Step title="Deploy to Web Server">
    <Tabs>
      <Tab title="Local Server">
        **Python HTTP Server:**
        ```bash
        python3 -m http.server 8000
        ```
        Open http://localhost:8000

        **Node.js (http-server):**
        ```bash
        npx http-server -p 8000
        ```

        **PHP Built-in Server:**
        ```bash
        php -S localhost:8000
        ```
      </Tab>

      <Tab title="Apache">
        Copy files to web root:
        ```bash
        sudo cp -r cotizaclima/* /var/www/html/
        ```

        **Optional .htaccess:**
        ```apache .htaccess
        # Enable compression
        <IfModule mod_deflate.c>
          AddOutputFilterByType DEFLATE text/html text/css text/javascript application/javascript
        </IfModule>

        # Cache static assets
        <IfModule mod_expires.c>
          ExpiresActive On
          ExpiresByType text/css "access plus 1 month"
          ExpiresByType application/javascript "access plus 1 month"
          ExpiresByType image/png "access plus 1 year"
        </IfModule>

        # Security headers
        Header set X-Content-Type-Options "nosniff"
        Header set X-Frame-Options "SAMEORIGIN"
        ```
      </Tab>

      <Tab title="Nginx">
        Create server block:
        ```nginx /etc/nginx/sites-available/cotizaclima
        server {
            listen 80;
            server_name cotizaclima.example.com;
            root /var/www/cotizaclima;
            index index.html;

            location / {
                try_files $uri $uri/ =404;
            }

            # Cache static assets
            location ~* \.(css|js|png|jpg|jpeg|gif|svg|webp)$ {
                expires 1y;
                add_header Cache-Control "public, immutable";
            }

            # Security headers
            add_header X-Content-Type-Options "nosniff" always;
            add_header X-Frame-Options "SAMEORIGIN" always;

            # Gzip compression
            gzip on;
            gzip_types text/css application/javascript;
        }
        ```

        Enable and restart:
        ```bash
        sudo ln -s /etc/nginx/sites-available/cotizaclima /etc/nginx/sites-enabled/
        sudo nginx -t
        sudo systemctl restart nginx
        ```
      </Tab>

      <Tab title="Firebase Hosting">
        Install Firebase CLI:
        ```bash
        npm install -g firebase-tools
        firebase login
        ```

        Initialize hosting:
        ```bash
        firebase init hosting
        ```

        **firebase.json:**
        ```json
        {
          "hosting": {
            "public": ".",
            "ignore": [
              "firebase.json",
              "**/.*",
              "**/node_modules/**"
            ],
            "headers": [
              {
                "source": "**/*.@(css|js)",
                "headers": [
                  {
                    "key": "Cache-Control",
                    "value": "max-age=31536000"
                  }
                ]
              }
            ]
          }
        }
        ```

        Deploy:
        ```bash
        firebase deploy --only hosting
        ```


### Offline Support

CotizaClima works offline by default, but you can add a Service Worker for full PWA support:

```javascript service-worker.js
const CACHE_NAME = 'cotizaclima-v1';
const urlsToCache = [
  '/',
  '/index.html',
  '/app.js',
  '/styles.css',
  'https://cdnjs.cloudflare.com/ajax/libs/jspdf/2.5.1/jspdf.umd.min.js'
];

self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then((cache) => cache.addAll(urlsToCache))
  );
});

self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request)
      .then((response) => response || fetch(event.request))
  );
});
```

Register in `index.html`:
```javascript
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('/service-worker.js');
}
```

## Troubleshooting

### Firebase Connection Issues

<AccordionGroup>
  <Accordion title="Status dot stays yellow/red">
    **Cause**: Database URL incorrect or rules deny access

    **Solution**:
    1. Verify `databaseURL` in `app.js:64` matches Firebase Console
    2. Check Database Rules allow public `.read: true`
    3. Open browser DevTools Network tab, look for failed requests
  </Accordion>

  <Accordion title="401 Unauthorized errors">
    **Cause**: Authentication required but not provided

    **Solution**:
    - For public read: Set rules to `.read: true`
    - For authenticated read: Login first, or provide `authToken` in config
  </Accordion>

  <Accordion title="Equipment loads demo data">
    **Cause**: Firebase connection failed, app fell back to demo

    **Solution**:
    ```javascript app.js:273
    // Check if DEMO_DATA is being used
    function loadDemoData() {
      allAC = DEMO_DATA;  // ← Fallback triggered
    }
    ```
    This is expected behavior when Firebase is unavailable.
  </Accordion>
</AccordionGroup>

### PDF Generation Issues

<AccordionGroup>
  <Accordion title="PDF button does nothing">
    **Cause**: jsPDF libraries not loaded

    **Solution**:
    ```html index.html:9-10
    <script src="https://cdnjs.cloudflare.com/ajax/libs/jspdf/2.5.1/jspdf.umd.min.js"></script>
    <script src="https://cdnjs.cloudflare.com/ajax/libs/jspdf-autotable/3.6.0/jspdf.plugin.autotable.min.js"></script>
    ```
    Verify these CDN links are accessible. For offline, download and host locally.
  </Accordion>

  <Accordion title="Logo missing in PDF">
    **Cause**: `refripro.png` not found

    **Solution**:
    ```javascript app.js:1161
    async function getPdfLogoSource() {
      // Add your logo file to root directory
      return 'data:image/png;base64,...';
    }
    ```
  </Accordion>
</AccordionGroup>

### Authentication Issues

<AccordionGroup>
  <Accordion title="Login fails with API key error">
    **Cause**: `FIREBASE_AUTH_CONFIG.apiKey` not set

    **Solution**:
    ```javascript firebase-auth.js:35
    function getApiKey() {
      const key = (FIREBASE_AUTH_CONFIG.apiKey || '').trim();
      if (!key || key === '') {
        throw new Error('Configura FIREBASE_AUTH_CONFIG.apiKey');
      }
      return key;
    }
    ```
    Get API key from Firebase Console → Project Settings → Web API Key
  </Accordion>

  <Accordion title="Token expired errors">
    **Cause**: Session token expired (default 1 hour)

    **Solution**: Automatic refresh implemented in `firebase-auth.js:61`
    ```javascript
    async function refreshIdToken(refreshToken) {
      // Automatically refreshes before expiry
    }
    ```
    User should not see this. If error persists, clear sessionStorage.
  </Accordion>
</AccordionGroup>

## Performance Optimization

### File Size Reduction

The app.js file is large (652KB) due to embedded demo data. For production:

```bash
# Minify JavaScript
npx terser app.js -o app.min.js -c -m

# Minify CSS
npx csso styles.css -o styles.min.css
```

Update references in `index.html`:
```html
<script src="app.min.js"></script>
<link rel="stylesheet" href="styles.min.css">
```

### CDN Acceleration

Host static assets on a CDN:

```html
<!-- Cloudflare, AWS CloudFront, etc. -->
<script src="https://cdn.example.com/cotizaclima/app.min.js"></script>
<link rel="stylesheet" href="https://cdn.example.com/cotizaclima/styles.min.css">
```

### Lazy Loading

For brand logos:
```html
<img src="assets/img/lg.svg" loading="lazy">
```
Already implemented in `app.js:521`.
