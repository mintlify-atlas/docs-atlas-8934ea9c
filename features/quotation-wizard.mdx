The quotation wizard is the core user interface of CotizaClima, guiding users through a 5-step process to create professional air conditioning quotations. The wizard ensures all required information is collected and validated before generating the final PDF.

## Overview

The wizard implements a progressive disclosure pattern, breaking down the complex quotation process into manageable steps:

<Steps>
  <Step title="Equipment Selection">
    Browse and filter the equipment catalog to select air conditioning units by brand, BTU capacity, and refrigerant type.
  </Step>
  
  <Step title="Service Type">
    Choose the type of service to provide: installation, maintenance, repair, equipment change, warranty visit, or quote only.
  </Step>
  
  <Step title="Accessories & Extras">
    Add optional accessories like piping kits, drainage systems, electrical cords, and mounting hardware.
  </Step>
  
  <Step title="Client Information">
    Enter customer details including name, RUT, contact information, and installation address with GPS support.
  </Step>
  
  <Step title="Summary & PDF">
    Review the complete quotation, adjust prices and discounts, then generate the professional PDF document.
  </Step>
</Steps>

```javascript
function updateWizardNav() {
  const step = window.currentStep || 1;
  const nextBtn = document.getElementById('btnWizardNext');
  
  let enabled = true;
  if (step === 1) {
    const hasItems = getSelectedEquipmentItems().length > 0;
    enabled = hasItems;
  }
  
  nextBtn.disabled = !enabled;
}
```

## Step-by-Step Workflow

### Step 1: Equipment Selection

The equipment page (index.html:72-113) includes:

**Search and Filters**
- Real-time search by brand or model (app.js:665-694)
- Brand dropdown populated from catalog
- BTU capacity filter (9K, 12K, 14K, 18K, 22K, 24K, 36K)
- Refrigerant type filter (R32, R410)
- "Más cotizados" button to sort by popularity

**Equipment Cards**

Each card displays (app.js:544-608):
- Brand logo (auto-loaded from assets/img/)
- Model name and type
- Refrigerant badge (R32/R410)
- Popularity ranking (fire icon with count)
- BTU capacity buttons
- Specifications: WiFi, warranty, installation kit

**Multi-Quantity Selection**

Users can select multiple units of the same or different models:

```javascript
function selectAC(idx, btu) {
  const key = equipKey(idx, Number(btu));
  selectedEquipments[key] = (selectedEquipments[key] || 0) + 1;
  updateStep1NextButton();
  renderACGrid();
}
```

<Note>
  The wizard validates that at least one equipment is selected before allowing progression to step 2.
</Note>

### Step 3: Accessories & Extras

The accessories page (index.html:137-156) displays a grid of optional add-ons:

**Accessory Categories** (loaded from Firebase or demo data):

- Ceiling mounts and supports
- Drainage kits
- Circuit breakers
- Refrigerant gas refills (R410/R32)

**Quantity Controls** (app.js:965-969):

### Step 4: Client Information

The client page (index.html:159-230) collects comprehensive customer data:

**Required Fields**:
- Full name (validated before PDF generation)
- Installation address

**Optional Fields**:
- RUT (Chilean tax ID, used for deduplication)
- Phone number
- Email address
- City and region
- Property type (house, apartment, office, commercial, warehouse/industrial)
- Additional notes

**GPS Address Autofill** (app.js:1721-1892):

The "Usar GPS" button uses browser geolocation and OpenStreetMap Nominatim reverse geocoding:

```javascript
function fillAddressFromGPS() {
  setGpsButtonLoading(true);
  setGpsStatus('Obteniendo ubicación...');
  
  navigator.geolocation.getCurrentPosition(
    async (position) => {
      const { latitude, longitude } = position.coords;
      // Reverse geocode using Nominatim API
      const address = buildStreetAddressWithFallback(data.address, data.display_name);
      // Auto-fill form fields
    },
    (error) => {
      setGpsStatus('No se pudo obtener ubicación', true);

}
```

<Note>
  Clients with a valid RUT are automatically deduplicated in the client directory. The system preserves client history and links all their quotations.
</Note>

### Step 5: Summary & PDF

The final step (index.html:234-347) displays a comprehensive review:

**Summary Cards**:
- Quote metadata (number, date, validity)
- Client information
- Equipment details with editable pricing
- Service details with editable pricing
- Accessories (if any)
- Discount percentage with calculated amount
- Grand total with IVA included

**Editable Pricing** (app.js:1118-1131):

```javascript
function recalcTotal() {
  const equipPrice = parseFloat(document.getElementById('sumEquipPrice').value) || 0;
  const servicePrice = parseFloat(document.getElementById('sumServicePrice').value) || 0;
  const extrasTotal = calculateAccessoriesTotal();
  const subtotal = equipPrice + servicePrice + extrasTotal;
  const discPct = parseFloat(document.getElementById('discountPct').value) || 0;
  const discAmt = subtotal * discPct / 100;
  const total = subtotal - discAmt;
  
  document.getElementById('grandTotal').textContent = fmtNum(Math.round(total));
}
```

**Actions**:
- Generate PDF (see [PDF Generation](/features/pdf-generation))
- Edit client data (returns to step 4)
- Save draft (persists to Firebase if logged in)
- Start new quotation (resets all state)


## Validation Rules

### Step 1 → Step 2
- At least one equipment must be selected
- Button state: `updateStep1NextButton()` checks `getSelectedEquipmentItems().length > 0`

### Step 2 → Step 3
- At least one service must be selected (either multi-quantity or single-select)
- Button state: `updateStep2NextButton()` validates service selection

### Step 3 → Step 4
- No validation required (accessories are optional)

### Step 4 → Step 5
- Client name is required for PDF generation
- Address is required for PDF generation
- Other fields are optional but recommended

## Responsive Design

The wizard adapts to different screen sizes:

- **Desktop**: Step indicator + bottom nav + floating controls
- **Mobile**: Bottom nav becomes primary navigation, step indicator compresses
- **Tablet**: Hybrid approach with all navigation methods visible

## Keyboard Shortcuts

No built-in keyboard shortcuts, but the wizard supports standard browser navigation:
- Tab/Shift+Tab for focus navigation
- Enter to activate buttons
- Arrow keys within select dropdowns

## Persistence

The wizard state can be saved and restored (see [Client Management](/features/client-management) for details):

- Draft quotes save automatically when "Guardar borrador" is clicked
- State is exported via `exportQuoteSnapshot()` (app.js:1560-1614)
- State is restored via `importQuoteSnapshot()` (app.js:1632-1709)
- All selections, quantities, notes, and client data are preserved

## Best Practices

<Accordion title="For Users">
  - Complete steps in order for the best experience
  - Use the search and filters to quickly find equipment
  - Add service notes with installation details (piping length, access issues)
  - Verify pricing in the summary before generating PDF
  - Save drafts frequently when working on complex quotations
</Accordion>

<Accordion title="For Developers">
  - Always update button states after modifying selections
  - Use `goToStep()` for navigation to ensure proper state updates
  - Validate required fields before allowing progression
  - Call `buildSummary()` when transitioning to step 5
  - Implement `window.currentStep` updates for external integrations
</Accordion>
