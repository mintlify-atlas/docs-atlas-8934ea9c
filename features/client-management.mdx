CotizaClima's client management system automatically maintains a deduplicated directory of customers, linking all quotations to client records for easy retrieval and relationship tracking.

## Overview

The client management system provides:

- **RUT-based deduplication**: Clients with the same RUT (Chilean tax ID) are automatically merged
- **Automatic synchronization**: Client data updates whenever a quote is saved
- **Quote association**: Bidirectional links between clients and their quotations
- **Client directory**: Browse all clients with their quote history
- **Non-invasive**: Works in the background without disrupting the quotation workflow

## RUT-Based Deduplication

### RUT Normalization

The Chilean RUT (Rol Único Tributario) serves as the unique identifier for deduplication (client-repo.js:38-41):

```javascript
function normalizeRut(rut) {
  if (!rut || typeof rut !== 'string') return '';
  return rut
    .replace(/\./g, '')      // Remove dots: "12.345.678-9" → "12345678-9"
    .replace(/\s/g, '')      // Remove spaces
    .toLowerCase()           // Lowercase check digit
    .trim();
}
```

**Examples**:
- `12.345.678-9` → `12345678-9`
- `12 345 678-9` → `12345678-9`
- `12.345.678-K` → `12345678-k`

### Client ID Resolution

Determines if a client can be deduplicated (client-repo.js:44-50):

```javascript
function resolveClientId(client) {
  if (!client) return null;
  const rut = normalizeRut(client.rut || '');
  
  // Require at least one digit + hyphen + digit/letter (minimum valid RUT)
  if (!rut || !/\d/.test(rut) || rut.length < 3) {
    return null;  // Cannot deduplicate without valid RUT
  }
  
  return rut;  // Use normalized RUT as Firebase key
}
```

<Note>
  Clients without a valid RUT are not persisted to the client directory. Their data still appears in the quotation but is not deduplicated across quotes.
</Note>

### Deduplication Behavior

When saving a quote with an existing RUT:

<Steps>
  <Step title="Check for existing client">
    Query Firebase: `GET /clients/{userId}/{rutKey}`
  </Step>
  
  <Step title="Preserve history">
    Keep original `createdAt` and existing `quoteIds`
  </Step>
  
  <Step title="Update fields">
    Overwrite name, phone, email, address with latest values
  </Step>
  
  <Step title="Add quote reference">
    Append new quote ID to `quoteIds` object
  </Step>
  
  <Step title="Save atomically">
    Use PUT to replace entire client record
  </Step>
</Steps>

Implementation (client-repo.js:113-156):

```javascript
async function syncClientForQuote(uid, idToken, clientObject, quoteId) {
  if (!clientObject || !clientObject.name) return null;
  const clientId = resolveClientId(clientObject);
  if (!clientId) return null;  // No RUT → skip sync
  
  try {
    const now = Date.now();
    
    // Fetch existing to preserve createdAt and quoteIds
    let existing = null;
    try {
      existing = await request(`clients/${uid}/${clientId}`, idToken, { method: 'GET' });
    } catch (e) {
      existing = null;
    }
    
    const quoteIds = Object.assign({}, existing ? (existing.quoteIds || {}) : {});
    if (quoteId) quoteIds[quoteId] = true;
    
    const payload = {
      name: clientObject.name || '',
      rut: clientObject.rut || '',
      rutKey: clientId,
      phone: clientObject.phone || '',
      email: clientObject.email || '',
      address: clientObject.address || '',
      city: clientObject.city || '',
      region: clientObject.region || '',
      property: clientObject.property || '',
      notes: clientObject.notes || '',
      createdAt: (existing && existing.createdAt) ? existing.createdAt : now,
      updatedAt: now,
      quoteIds: quoteIds
    };
    
    await request(`clients/${uid}/${clientId}`, idToken, {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(payload)
    });
    
    return clientId;
  } catch (e) {
    // Silently fail — this is a background operation
    return null;
  }
}
```

## Automatic Synchronization

Client data synchronizes automatically during quote persistence (quote-persistence.js):

```javascript
async function persistCurrentQuote(status = 'draft') {
  const snapshot = window.CotizPersistenceBridge.exportQuoteSnapshot(status);
  const uid = window.FirebaseAuthService.getCurrentUser().uid;
  const idToken = await window.FirebaseAuthService.getValidIdToken();
  
  // Save quote first
  const result = await window.QuotesRepo.saveQuote(
    uid, 
    idToken, 
    snapshot, 
    activeQuoteId
  );
  
  // Then sync client in background (fire-and-forget)
  window.ClientRepo.syncClientForQuote(
    uid,
    idToken,
    snapshot.client,
    result.id
  ).catch(() => {});
  
  return result;
}
```

<Note>
  Client sync is fire-and-forget. If it fails, the quote still saves successfully. This prevents network issues from blocking the user workflow.
</Note>

### Adding Quote to Existing Client

Non-destructive PATCH operation (client-repo.js:99-106):

```javascript
async function addQuoteToClient(uid, idToken, clientId, quoteId) {
  if (!clientId || !quoteId) return;
  
  await request(`clients/${uid}/${clientId}/quoteIds`, idToken, {
    method: 'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ [quoteId]: true })
  });
}
```

<Note>
  PATCH only adds the new quote ID without overwriting existing ones. This is safer than GET → modify → PUT.
</Note>

async function listClients(uid, idToken) {
  const raw = await request(`clients/${uid}`, idToken, { method: 'GET' });
  if (!raw || typeof raw !== 'object') return [];
  
  return Object.entries(raw)
    .map(([id, data]) => ({ id, ...data }))
    .sort((a, b) => (a.name || '').localeCompare(b.name || '', 'es'));
}

## GPS Address Autofill

The "Usar GPS" button helps fill address fields automatically (app.js:1721-1892):

```javascript
function fillAddressFromGPS() {
  setGpsButtonLoading(true);
  setGpsStatus('Obteniendo ubicación...');
  
  navigator.geolocation.getCurrentPosition(
    async (position) => {
      const { latitude, longitude } = position.coords;
      
      // Reverse geocode using OpenStreetMap Nominatim
      const response = await fetch(
        `https://nominatim.openstreetmap.org/reverse?` +
        `format=json&lat=${latitude}&lon=${longitude}&addressdetails=1`
      );
      const data = await response.json();
      
      // Fill form fields
      const addressResult = buildStreetAddressWithFallback(data.address, data.display_name);
      document.getElementById('clientAddress').value = addressResult.value;
      
      const city = pickCityFromAddress(data.address);
      document.getElementById('clientCity').value = city;
      
      const region = mapRegionFromState(data.address.state);
      if (region) document.getElementById('clientRegion').value = region;
      
      setGpsStatus('Dirección obtenida correctamente', false);
    },
    (error) => {
      setGpsStatus('No se pudo obtener ubicación', true);
      setGpsButtonLoading(false);
    }
  );
}
```

<Note>
  GPS functionality requires HTTPS and user permission. The feature degrades gracefully if unavailable.
</Note>

## Error Handling

The client system is designed to never block the quotation workflow:

### Silent Failures

```javascript
try {
  await ClientRepo.syncClientForQuote(uid, idToken, client, quoteId);
} catch (e) {
  // Silent failure — don't interrupt user flow
  console.warn('Client sync failed:', e);
}
```

### Validation

- **Name required**: Cannot save quote without client name
- **RUT optional**: Quotes without RUT still save, but client won't be deduplicated
- **Format validation**: RUT format not enforced (accepts any string)

### Network Issues

- Quote saves even if client sync fails
- Client data still appears in PDF
- Next quote save will retry sync

## Best Practices

<Accordion title="For Users">
  - Always enter the client's RUT to enable deduplication
  - Use consistent RUT formatting (system normalizes automatically)
  - Update client info in latest quote (changes propagate to directory)
  - Review client directory periodically to merge duplicates (manual)
</Accordion>

<Accordion title="For Developers">
  - Never block on client operations (use fire-and-forget)
  - Preserve `createdAt` and `quoteIds` when updating
  - Use PATCH for adding quote IDs (safer than PUT)
  - Normalize RUT before using as Firebase key
  - Handle missing RUT gracefully (skip sync, don't error)
</Accordion>

## Data Privacy

- Client data is scoped to user ID (no cross-user access)
- Firebase rules should enforce: `clients/{userId}` only readable/writable by `{userId}`
- No PII is logged to console or analytics
- Client data remains in Firebase (not cached in localStorage)
