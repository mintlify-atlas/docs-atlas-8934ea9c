Get up and running with CotizaClima in minutes. This guide walks you through creating your first professional HVAC quote from equipment selection to PDF generation.

## What You'll Build

By the end of this quickstart, you'll have:
- Selected air conditioning equipment from the catalog
- Configured installation services
- Added optional accessories
- Entered client information
- Generated a professional PDF quote

## Prerequisites

All you need is:
- A modern web browser (Chrome, Firefox, Safari, Edge)
- The CotizaClima application files (see [Installation](/installation))
- 2 minutes of your time

<Note>
  CotizaClima works 100% offline by default. No internet connection required to create quotes.
</Note>

## Create Your First Quote

<Steps>
  <Step title="Open the Application">
    Navigate to `index.html` in your web browser. You'll see the CotizaClima interface with a 5-step wizard:

    1. **Equipo** (Equipment)
    2. **Servicio** (Service)
    3. **Extras** (Accessories)
    4. **Cliente** (Client)
    5. **Resumen** (Summary)

    The wizard automatically tracks your progress through the top step indicator.
  </Step>

  <Step title="Select Equipment">
    Filter and select air conditioning units:

    ```html
    <!-- Search by brand or model -->
    <input id="searchInput" placeholder="Buscar marca o modelo…">
    
    <!-- Filter by capacity, refrigerant, brand -->
    <select id="filterBTU">  <!-- 9K, 12K, 18K, 24K, 36K -->
    <select id="filterRef">  <!-- R32, R410 -->
    <select id="filterBrand"> <!-- All brands from catalog -->
    ```

    **To add equipment:**
    - Browse the equipment grid (loaded from Firebase or demo data)
    - Click a BTU capacity button (9K, 12K, 18K, etc.)
    - Use **+/−** buttons to adjust quantity
    - Multiple equipment types can be selected

    <Info>
      The "Más cotizados" (Most Quoted) button sorts equipment by popularity when logged in with Firebase Authentication.
    </Info>

    Click the **→** (forward) button or navigate to **Servicio** when ready.
  </Step>

  <Step title="Choose Service Type">
    Select the type of service from 6 options:

    <CodeGroup>
    ```javascript Service Types
    const SERVICES = [
      { 
        id: 'instalacion',
        name: 'Instalación nueva',
        desc: 'Primera instalación del equipo',
        price: 100000
      },
      { 
        id: 'mantencion',
        name: 'Mantención',
        desc: 'Limpieza y revisión general',
        price: 30000
      },
      { 
        id: 'cambio',
        name: 'Cambio + instalación',
        desc: 'Retiro equipo antiguo + nuevo',
        price: 120000
      },
      { 
        id: 'reparacion',
        name: 'Reparación',
        desc: 'Diagnóstico y reparación',
        price: 10000
      },
      { 
        id: 'garantia',
        name: 'Visita garantía',
        desc: 'Revisión por garantía vigente',
        price: 0
      },
      { 
        id: 'presupuesto',
        name: 'Solo presupuesto',
        desc: 'Cotización sin instalación',
        price: 0
      }
    ];
    ```
    </CodeGroup>

    **Multi-service support:**
    - Installation and maintenance services support multiple quantities
    - Unit prices can be edited inline
    - Service notes can be added (appear in final PDF)

    The app automatically preloads 1 installation per equipment selected.
  </Step>

  <Step title="Add Accessories (Optional)">
    Add extra items from the accessories catalog:

    - **Cañerías** (Piping kits) - for 9K, 12-18K, 24K units
    - **Eléctrico** (Electrical) - power cords, breakers
    - **Instalación** (Installation) - ceiling mounts, drainage kits
    - **Gas** (Refrigerant) - R410, R32 refills
    - **Mantención** (Maintenance) - cleaning services

    Use the **+/−** quantity controls for each accessory. Skip this step if no extras are needed.
  </Step>

  <Step title="Enter Client Information">
    Fill in client details:

    ```javascript Required Fields
    {
      name: 'Juan Pérez González',      // Required
      address: 'Calle Principal 123'    // Required
    }
    ```

    ```javascript Optional Fields
    {
      rut: '12.345.678-9',
      phone: '+56 9 1234 5678',
      email: 'correo@ejemplo.cl',
      city: 'Curicó',
      region: 'Maule',
      property: 'Casa',  // Casa, Departamento, Oficina, etc.
      notes: 'Piso 2, acceso por escalera'
    }
    ```

    <Info>
      The **Usar GPS** button autofills address and city from your current location using the browser's geolocation API.
    </Info>

    When Firebase Authentication is enabled, clients with RUT are automatically saved to the clients database for future quotes.
  </Step>

  <Step title="Review and Generate PDF">
    The summary page displays:

    - **Quote metadata**: Number (CL-2026-0001 format), date, validity (15 days)
    - **Client details**: All entered information
    - **Equipment**: Model, BTU, refrigerant, editable price
    - **Service**: Type, quantity, editable price
    - **Accessories**: Line items with quantities
    - **Discount**: Percentage-based discount field
    - **Total**: Final amount with IVA (tax) included

    **Edit prices** directly in the summary:
    ```html
    <input type="number" id="sumEquipPrice" oninput="recalcTotal()">
    <input type="number" id="sumServicePrice" oninput="recalcTotal()">
    <input type="number" id="discountPct" oninput="recalcTotal()">
    ```

    Click **Generar PDF · Descargar** to create and download the quote as a PDF file.
  </Step>
</Steps>

## What Happens Next?

<CardGroup cols={2}>
  <Card title="Quote Numbering" icon="hashtag">
    Quotes are auto-numbered in `CL-YYYY-####` format. The sequence increments automatically and syncs with Firebase when authenticated.
  </Card>
  
  <Card title="PDF Generation" icon="file-pdf">
    PDFs are generated client-side using jsPDF with:
    - Professional branding header
    - Equipment specifications table
    - Service details with inclusions
    - Terms and warranty information
  </Card>
  
  <Card title="Save as Draft" icon="floppy-disk">
    Click **Guardar borrador** to save the quote to Firebase. Requires authentication. Drafts can be reopened from the history page.
  </Card>
  
  <Card title="Start New Quote" icon="rotate-right">
    Click **Nueva cotiz.** to reset all fields and start fresh. Quote number increments automatically.
  </Card>
</CardGroup>

## Tips for First-Time Users

<Warning>
  **Required fields validation**: The forward navigation button (→) is disabled until:
  - Step 1: At least one equipment is selected
  - Step 2: At least one service is selected
  - Step 4: Client name and address are filled
</Warning>

<Info>
  **Keyboard-friendly**: All form fields support standard keyboard navigation. Use Tab to move between fields.
</Info>

## Troubleshooting

### Equipment List Not Loading

If you see "Cargando equipos…" indefinitely:

1. Check if `FIREBASE_CONNECTION.databaseURL` is configured in `app.js`
2. Verify Firebase Realtime Database rules allow public read access
3. The app will fallback to demo data automatically after a few seconds

```javascript app.js:64
const FIREBASE_CONNECTION = {
  databaseURL: 'https://your-project-default-rtdb.firebaseio.com/',
  node: 'air_conditioners',
  authToken: '' // Optional
};
```

### PDF Not Downloading

Ensure the jsPDF libraries are loaded:

```html index.html:9
<script src="https://cdnjs.cloudflare.com/ajax/libs/jspdf/2.5.1/jspdf.umd.min.js"></script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/jspdf-autotable/3.6.0/jspdf.plugin.autotable.min.js"></script>
```

### Quote Number Not Incrementing

Quote numbers use localStorage for persistence:

```javascript app.js:408
function getLocalQuoteCounterKey(datePart) {
  return `cotiz_quote_counter_${datePart}`;
}
```

Clear browser cache if the counter gets stuck, or authenticate with Firebase for cloud-synced numbering.
